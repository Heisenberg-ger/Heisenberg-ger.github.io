Docker，WSL报错修复
Created2023-10-31|报错修复
在我新加了一块硬盘之后，发现Docker报错无法打开，经排查后发现wsl安装不上，并报错： 在尝试过包括重装，关闭可选虚拟化功能等方法后均无效，最后发现是VMware出了问题。 解决方法： 卸载VMware后重装，如果你和我一样把原来的盘给合并了，那么就先新增一个原来一样的盘符，如D: 打开磁盘管理-右键压缩卷-分配少一点空间（几个g）-继续-新建卷-新建一样的盘符-一直下一步。 然后卸载VMware，此时再安装wsl2就可以成功了，如运行wsl –set-version Ubuntu-20.04 2，安装完成后docker也就能打开了

Ubuntu传递文件
Created2023-10-13|教程
在安装完Ubuntu后想传输文件非常的麻烦 网上查阅教程大多都说安装完VMware tools 后便可自由拖拽文件，但安装完后并不行 查阅发现从Ubuntu14.04开始open-vm-tools 代替了官方 VMware Tools，所以应该改安装open-vm-tools。 步骤1.打开Ubuntu的设置页面，打开关于-软件更新，更改下载自：来自中国的服务器，关闭窗口，重启Ubuntu 2.打开终端，输入sudo apt-get install open-vm-tools-desktop，根据提示完成安装后应该能观察到页面自动调整了大小，此时说明安装成功。重新启动Ubuntu便可拖拽传输文件
